Estimate time series mortality damages (copy cat version of Climate Team)

Calculate damages using FAIR anomalies (either control or pulse). This funtion calculates a base period average and calculate temperature anomalies using that climatology. These anomalies are later used to estimate damages using the betas from the menu option.

Parameters

  • fair_temperature (xr.DataArray or xr.Dataset) – xarray object containing FAIR temperatures. These can be either from control or pulses runs.

  • betas (xr.DataSet) – An xarray object containing the fit coefficients from the yearly damage functions

  • anomalies (bool) – If True do not calculate temperature anomalies. Then fair_temperatures must be anomalies.

scc_multiverse.fair.estimations.estimate_damages_betas(damage_function, type_estimation, return_preds=True, base_year=2010, interp_year=2085, year_start_pred=2100, year_end_pred=2300)[source]¶

Estimate global damages betas using global climate anomalies

This function will calculate a quadratic fit of damages across IAM/GCM/RCP using model temperature anomalies from CIMP5. The model to be estimated is the following:

..math::

Damages_{gcm, rcp, iam} = eta_{0} + eta_1 * temperature + eta2 * temperature^{2} +

arepsilon

This model is estimated for each year in our timeframe (2010 to 2099) and using a rolling window across time (5-yr by default). For the years between 2099 and 2300, we use a linear extrapolation of the quadratic fit of damages starting on interp_year_start (set as 2085 as the default). For this extrapolation we use a linear temporal interaction with the pre-2100 estimated damages.

Regression coefficients for each model are temporally stored and then used to estimate predicted damages damages by each quarter-degree celsius (C). Hence, we will return a pd.DataFrame with predicted damages for each year over an array of 40 different temperatures.

Options are available to change the size of the rolling window, as well as the prediction years and base years.

damage_functionpd.DataFrame

Model/Batch damage realizations indexed by temperature anomalies.

type_estimationstr

Type of model use for damage fitting. ols and quantile are the two available options.

return_predsbool

If True return a data frame with predictions. Otherwise, just return yearly betas.

base_yearint

Base year to use interpolation. 2010 is the set default.

interp_yearint

Year pre-2100 to fit damages used for linear extrapolation. i.e. If interp_year=2085, data from years 2085 to 2099 will be used to extrapolate data to years post-2100. 2085 is the default.

year_start_predint

Start interpolation. Default is 2100

year_end_predint

End interpolation. Default is 2300

pd.DataFrame

A DataFrame containing yearly predictions of damages per each quarter-degree starting in 2010 and ending in year_end_pred. If return_preds=False, the the function will return a DataFrame with estimation betas per year.

scc_multiverse.fair.load_fair module¶

We can run FAIR with the CO$_2$ emissions and non-CO$_2$ forcing from the four representative concentration pathway scenarios. To use the emissions-based version specify `useMultigas=True` in the call to `fair_scm()`.

By default in multi-gas mode, volcanic and solar forcing plus natural emissions of methane and nitrous oxide are switched on.

We can compute the SCC by adding an additional pulse in CO2 emissions to the RCP trajectory.

This study uses a 1 Gt C emissions pulse. You can change the pulse amount by modifying the PULSE_AMT variable.

scc_multiverse.fair.utils.load_parameters(name, cache=True, token=True, data_home=None, **kwargs)[source]¶

Retrieve parameter files from private data repository

This function provides a cleaner access to remotely stored data for FAIR parameters. Instead of relying on the user file system, this function requests parameter NetCDF files from a private Gitlab repo.

Parameters

  • name (str) – Name of parameter dataset

  • token (str) – Gitlab Private personal token. Default is None. Will look for the environment variable GITLAB_PERSONAL_TOKEN.

  • cache (boolean, optional) – If True, try to read from the cached data in the environment variable FAIR_DATA.

  • data_home (str, optional) – Directory to cache data. fair-parameter-data by default

  • kwargs (dict, optional) – Additional values passed to xarray.open_dataset.

Returns

xr_array

Return type

xr.Dataset
